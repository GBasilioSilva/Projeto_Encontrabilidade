Introdução

Este artigo tem por finalidade analisar as potencialidades existentes na arquitetura de desenvolvimento de software da Microsoft chamada Asp.Net Core. Esta arquitetura é uma reescrita baseada em seus frameworks anteriores, porém, com o viés de ser Open Source e trazer algumas novidades em relação ao ambiente desenvolvimento. Baseado em bibliotecas C# (CSharp), essa arquitetura veio com a pretensão de inovar o mercado de desenvolvimento de software.

Levando em consideração o panorama do mercado de desenvolvimento de softwares, a escolha da ferramenta se torna fundamental. Pois então, qual das existentes selecionar, sendo que dentre elas se encontra o Asp.Net Core? Quais são suas principais concorrentes e os seus respectivos aspectos mais relevantes? E porque a relevância desta arquitetura? Com base em análises, as respostas serão construídas durante o percurso deste artigo.

O mercado de Desenvolvimento

Um dos ditados que definem a realidade do mercado de desenvolvimento é “tempo é dinheiro”. Quanto mais rápido se desenvolve uma solução, melhor. A questão é a quantidade de ferramentas oferecidas para isto. São muitas ferramentas para o mesmo trabalho, porém, cada uma com suas peculiaridades. Então, como anda o ASP.NET Core?

Fundamentação Teórica


ASP.NET Core, O que é? Onde vive? Como surgiu?

MICROSOFT (2001) afirma que “Microsoft .NET é a estratégia da Microsoft para distribuir softwares como serviço. Inclui a infraestrutura .NET e as ferramentas para criar e operar uma nova geração de serviços, a experiência do usuário .NET para possibilitar rich clientes, serviços de building blocks de construção da .NET e softwares de dispositivo .NET para possibilitar uma nova geração de dispositivos de Internet inteligentes.”

Inicialmente, as aplicações desenvolvidas só eram suportadas para a plataforma Windows. Com a evolução da internet e das multiplataformas derivadas, a Microsoft investiu no desenvolvimento e na ampliação do .NET. Então, em agosto de 2002 surgiu o ASP.NET como uma evolução do seu predecessor ASP (Active Server Pages) que ganhou popularidade rapidamente. Na época, 21% das aplicações Web desenvolvidas eram em ASP.NET e concorria diretamente com o JAVA e o PHP. Em sua primeira versão, foi introduzido o chamado “Web Forms” ou formulários web em que, levou para a internet as mesmas vantagens que ele possuía para desenvolvimento desktop. Também agregou o tratamento de aspectos HTML, HTTP e manutenção do estado da aplicação.

Conclusão


Hoje, tem-se uma arquitetura totalmente reescrita, com novas funcionalidades Web, Open Source e com uma capacidade de melhoria contínua muito alta. Também se tem a possibilidade de utilizá-lo integrado ao Visual Studio, uma IDE da Microsoft ou, pode-se usar coms ua IDE favorita através de terminal de comando. Não se pode negarque há um exponencial crescente em voltadesta arquitetura pelos motivos já ditos. É uma revolução da Microsoft e você decide oquanto é relevante usar em seus projetos.
